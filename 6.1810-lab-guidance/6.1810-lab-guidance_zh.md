# 实验室指引

## 作业难度说明

每份作业均标有难度等级，说明如下：

**简单**：一小时以内。此类练习通常作为后续练习的热身，帮助熟悉基本概念。

**中等**：一至两小时。

**困难**：两小时以上。此类练习往往不需要大量代码，但代码本身极难写正确。

以上时间仅为粗略参考。部分选做作业我们尚无标准答案，难度评估亦属估计。若你发现自己在某道题上花费的时间远超预期，欢迎在 Piazza 上提问，或前来参加答疑课时。

实验题目通常无需大量代码（数十行至数百行不等），但概念上颇为复杂，细节至关重要。因此，动笔之前，请务必完成指定阅读，通读相关文件，并查阅必要的参考文档（RISC-V 手册等均可在参考页面找到）。建议将实现过程分解为若干小步骤——作业本身通常也会给出分解思路——并在推进至下一步前验证当前步骤是否正确。

**注意**：请勿在截止日期前夜才开始动手。将工作分散至多日完成，效率往往更高。操作系统内核中的缺陷可能以出人意料的方式显现，理解并修复它有时需要大量的思考与耐心的调试。

## 调试建议

### 掌握 C 语言与指针

请确保你对 C 语言和指针有扎实的理解。Kernighan 与 Ritchie 所著的《C 程序设计语言（第二版）》是简明扼要的参考读物。建议阅读相关示例代码，并仔细理解其输出结果的成因。

以下几种指针惯用法尤为值得牢记：

- 若 `int *p = (int*)100`，则 `(int)p + 1` 与 `(int)(p + 1)` 的结果不同：前者为 `101`，后者为 `104`。将整数加到指针上时（如后者），整数会隐式乘以指针所指对象的大小。
- `p[i]` 等价于 `*(p+i)`，表示 `p` 所指内存中第 `i` 个对象。上述加法规则使得这一定义在对象大于一字节时仍能正确工作。
- `&p[i]` 等价于 `(p+i)`，得到 `p` 所指内存中第 `i` 个对象的地址。

尽管多数 C 程序无需在指针与整数之间相互转换，但操作系统代码中此类操作极为常见。每当遇到涉及内存地址的加法运算，请自问：这是整数加法还是指针加法？被加的值是否经过了适当的乘法处理？

### 及时提交，保留检查点

若某道练习已部分完成，请通过提交代码的方式保存当前进度。一旦后续改动引入了新问题，便可回退至此检查点，再以更小的步伐向前推进。如需了解 Git 的使用方法，可参阅 Git 用户手册，或阅读面向计算机科学学生的 Git 概述文章。

### 插入打印语句

若代码未能通过某项测试，请务必弄清原因。通过插入打印语句，逐步厘清程序的执行过程。

如果打印输出过多难以浏览，可以在 `script` 命令（运行 `man script` 查看说明）中执行 `make qemu`，将所有控制台输出记录至文件，再进行搜索。使用完毕后，别忘了退出 `script`。

### 使用 GDB 单步调试

打印语句在多数情况下已足够强大，但有时需要逐行执行汇编代码，或检查栈上的变量。在 xv6 中使用 gdb 的方法如下：

1. 在一个窗口中运行 `make qemu-gdb`；
2. 在另一个窗口中运行 `gdb-multiarch`（或 `riscv64-linux-gnu-gdb`、`riscv64-unknown-elf-gdb`）。若使用 Athena，请确保两个窗口位于同一台 Athena 机器上；
3. 设置断点，然后输入 `c`（continue）继续执行，xv6 将运行至断点处停止。

详见《使用 GNU 调试器》一文中的 GDB 使用技巧。若启动 gdb 时出现形如 `'warning: File ".../.gdbinit" auto-loading has been declined'` 的警告，请按照提示编辑 `~/.gdbinit`，添加 `add-auto-load-safe-path ...`。

### 查阅汇编文件

若需查看编译器为 xv6 内核生成的汇编代码，或查找某个内核地址对应的指令，可参阅 `kernel/kernel.asm` 文件——该文件由 Makefile 在编译内核时自动生成。Makefile 同样会为所有用户程序生成对应的 `.asm` 文件。

### 处理内核异常

若内核发生意外故障（例如访问了非法内存地址），它将打印一条错误信息，其中包含崩溃时的程序计数器值（`sepc`）。你可以在 `kernel.asm` 中搜索该值所在的函数，也可以运行 `addr2line -e kernel/kernel <pc值>`（运行 `man addr2line` 查看详情）。

若需获取调用栈回溯，请重新使用 gdb：在一个窗口运行 `make qemu-gdb`，在另一个窗口运行 gdb，在 `panic` 处设置断点（`b panic`），然后输入 `c` 继续。当内核触发断点时，输入 `bt` 即可获取回溯信息。

### 处理内核挂起

若内核挂起（例如因死锁），可使用 gdb 定位挂起位置：在一个窗口运行 `make qemu-gdb`，在另一个窗口运行 gdb，输入 `c` 继续。当内核看起来已挂起时，在 `qemu-gdb` 窗口按 `Ctrl-C`，然后输入 `bt` 获取调用栈。

### 使用 QEMU 监控器

QEMU 内置一个监控器，可用于查询模拟机器的状态。按下 `Ctrl-A C` 即可进入（其中 `C` 代表 console）。一个特别实用的监控器命令是 `info mem`，可用于打印页表。如需指定查看哪个核心，可使用 `cpu` 命令切换，或在启动时通过 `make CPUS=1 qemu` 将核心数限制为一个。

# 6.1810 课程工具安装指南

本课程需要以下 RISC-V 版本的工具：QEMU 7.2+、GDB 8.3+、GCC 和 Binutils。

如果安装过程中遇到问题，请前往答疑时间或在 Piazza 上发帖，我们很乐意提供帮助。

---

## Debian / Ubuntu

```bash
sudo apt-get install git build-essential gdb-multiarch qemu-system-misc gcc-riscv64-linux-gnu binutils-riscv64-linux-gnu
```

> **注意**：需要运行 Ubuntu 24 或更新版本，才能通过 `apt-get` 安装足够新版本的 QEMU。

---

## Arch Linux

```bash
sudo pacman -S riscv64-linux-gnu-binutils riscv64-linux-gnu-gcc riscv64-linux-gnu-gdb qemu-emulators-full bc git base-devel
```

## Windows

建议 Windows 用户选择以下方式之一：

- 在本机上安装 Linux
- 使用 **WSL 2**（Windows Subsystem for Linux 2）

同时推荐安装 **Windows Terminal**，以替代 PowerShell 或命令提示符。

### 配置 WSL 2

1. 确认已安装 Windows Subsystem for Linux
2. 从 Microsoft Store 添加 **Ubuntu 24.04**
3. 启动 Ubuntu 并开始使用

> **重要**：请确保运行的是 WSL **版本 2**，WSL 1 与 6.1810 的实验不兼容。在 Windows 终端中运行以下命令确认版本：
> ```
> wsl -l -v
> ```

### 安装所需软件

```bash
$ sudo apt-get update && sudo apt-get upgrade
$ sudo apt-get install git build-essential gdb-multiarch qemu-system-misc gcc-riscv64-linux-gnu binutils-riscv64-linux-gnu
```

在 Windows 中，可以通过 `\\wsl$\` 目录访问所有 WSL 文件。例如，Ubuntu 20.04 的主目录路径为：

```
\\wsl$\Ubuntu-20.04\home\<用户名>\
```

## 运行 Linux 虚拟机

如果当前操作系统不便于直接安装 RISC-V 工具，可以运行 Linux 虚拟机并在其中安装。

### 第一步：安装虚拟化平台

推荐使用 **VirtualBox**（支持 Mac、Linux、Windows，免费）。

### 第二步：获取 Linux 镜像

推荐使用 **Ubuntu Desktop**，下载后会得到类似 `ubuntu-20.04.3-desktop-amd64.iso` 的文件。在虚拟化平台中创建新的 64 位虚拟机，并使用该镜像作为启动盘即可。

## macOS

### 第一步：安装开发者工具

```bash
$ xcode-select --install
```

### 第二步：安装 Homebrew

```bash
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 第三步：安装 RISC-V 编译器工具链

```bash
$ brew tap riscv/riscv
$ brew install riscv-tools
```

若出现以下错误：

```
Error: Refusing to load formula riscv/riscv/riscv-tools from untrusted tap riscv/riscv.
```

原因是 Homebrew 默认不信任第三方 tap，需手动授权。依次执行以下命令：

```bash
# 1. 信任 tap
brew trust riscv/riscv

# 2. 重新安装 riscv-tools
brew install riscv-tools
```

安装过程涉及编译多个组件，耗时较长，请耐心等待。

> Homebrew 的 formula 可能不会自动链接到 `/usr/local`，需要手动将以下路径添加到 shell 配置文件（如 `~/.bashrc`）：
>
> ```bash
> PATH=$PATH:/usr/local/opt/riscv-gnu-toolchain/bin
> ```

### 第四步：安装 QEMU

```bash
$ brew install qemu
```

## 验证安装

参照第一个实验的说明，尝试编译并运行 xv6。也可以通过以下命令手动检查：

**检查 QEMU 版本**（应为 7.2.0 或更高）：

```bash
$ qemu-system-riscv64 --version
QEMU emulator version 7.2.0
```

**检查 GCC 版本**（以下任意一个即可）：

```bash
$ riscv64-linux-gnu-gcc --version
riscv64-linux-gnu-gcc (Debian 10.3.0-8) 10.3.0

$ riscv64-unknown-elf-gcc --version
riscv64-unknown-elf-gcc (GCC) 10.1.0

$ riscv64-unknown-linux-gnu-gcc --version
riscv64-unknown-linux-gnu-gcc (GCC) 10.1.0
```
